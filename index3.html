О программе

Программа изготовлена в компании: ООО Битник.
Программа является кассовым терминалом с возможностью:

  • Производить настройки подключаемого кассового аппарата.

  • Выбирать товар для продажи через сканер шрих кода

  • Пробивать чеки текущей продажи За наличный расчет.

  • Пробивать чеки текущей продажи Эквайрнг (Сбер. Банк).

  • Осуществлять возврат товара по чеку в контексте текущей смены.

  • Учет остатков в торговой точке на момент продажи.

  • Мониторинг исторических данных о продажи в разрезе продавца, торговой точки.

  • Принимать служебные распоряжения.

  • Отправлять различные заявки.

Программа написана на Java JavaFx .
Программа может быть развернута на Windows и Unix подобных операционных системах,
разрядностью 32-64 бит. Программа может эксплуатироваться на компьютере с типом мониора - сенсорный экран (touchscreen)
Программа поддерживает контрольно кассовое оборудование производителя АТОЛ

Установка программы

На Linux (Ubuntu) программа ставится запуском скрипта – установщика с правами администратора.
Кассовый аппарат (ККM) подключается через USB порт. Пинпад подключается через USB порт, настроек в программе не требует, пинпад должен быть настроен сотрудниками банка.
Сканер штрих кода подключается через USB порт в режиме клавиатуры, настроек не требует

Внимание: В ККM, должен быт выставлен Режим обмена – USB.

На ОС Windovs программа ставится вручную.
Ставится драйвер ККM, настраивается режим обмена через Интернет.
Установка драйвера ккм.
Настройка службы EoU.

Старт программы

Старт программы происходит автоматически при загрузке компьютера
Так же можно осуществить старт через ярлык на рабочем столе.

Внимание: Перед загрузкой системы или старте программы, кассовый аппарат должен быть подключен.
Если этого не произошло, выполнить действие поздне, после подключения ККМ к ком-ру произвести: КАССА -> СОСТОЯНИЕ КАССЫ.

Во время старта, программа обращается к кассовому аппарату, за данными о текущей смене. Главные настройки программы отображаются в нижнем баре окна программы:

  • Точка: идентификатор предприятия (торговой точки) цифровой тип.

  • Версия: Текущая версия программы.

  • Профиль: Профиль программы, к какому серверу обращается программа (определяется в коде программы)

  • Эквайринг: Выключен или банк оператора эквайринга.

  • Выручка: Текущая выручка, берется с ККМ, при каждом запросе программы к ККМ,в разрезе смены ККМ.

  • Сменный итог:Текущий сменный итог, берется с ККМ, при каждом запросе программы к ККМ,в разрезе смены ККМ.

  • Наличность в Д.Я.: Наличность в денежном ящике ККМ, берется с ККМ, при каждом запросе программы к ККМ,в разрезе смены ККМ.

  • Б. Нал: Сумма текущий безналичных операций в разрезе смены ккм, считается программой на основании сохраненных данных.

  • Смена: Номер смены в ККМ.

  • Статус смены ККМ: Смена открыта - Смена закрыта.

Вход пользователя

Список всех доступных пользователей, (включая администратора), программа получает с сервера по ручному запросу.
Так же, в этом запросе, приходят все доступные продукты для продажи и их реквизиты.
При первом запуске программы, база данных пустая.
Первый запуск должен осуществить установщик программы, по специальному паролю который вшит в программный код.
После удачного входа, он должен обновить базу данных с сервера, в этих данных будет и пароль
системного администратора, который будет обслуживать программу.
Вход пользователя и администратора осуществляется через модальное окно: ДЕЙСТВИЯ -> ВХОД.

Внимание: Mожет случиться так, что пароль входящего продавца может отсутствовать в базе данных.
Для этого набираем 911911 и входим с правами только обновления базы данных.
После обновления: ДЕЙСТВИЯ -> ОБНОВИТЬ ДАННЫЕ делаем выход: ДЕЙСТВИЯ -> ВЫХОД
Производим вход: ДЕЙСТВИЯ -> ВХОД и пытаемся ввести пароль полученный у администратора.
Если ничего не получается - обращаемся к администратору.

Существует маленькое удобство, пароль можно вводить через сканер штрих кода.

Внимание: При удачном входе , автоматически происходит заливка в ККМ имени продавца (печатается на чеке)
если кассовый аппарат был не подключен или произошел сбой при передаче данных, ввод имени продавца осуществить вручную: КАССА->СМЕНА КАССИРА ККМ.
Обязательно убедиться, что смена пользователя в ККМ произошла удачно - вызвать окно состояния ККМ: КАССА -> СОСТОЯНИЕ КАССЫ. Поле: Кассир

После этого можно производить дальнейшие действия.

Открытие Закрытие смены

В начале рабочего дня продавец обязан открыть смену в ККМ, в конце рабочего дня произвести закрытие смены (Снятие Z отчета).
Открытие смены включает в себя:

  • Обновление локальных данных с сервера ДЕЙСТВИЯ -> ОБНОВИТЬ ДАННЫЕ

  • Открытие смены в ККМ: КАССА -> ОТКРЫТИЕ СМЕНЫ, происходит открытие смены в ККМ, печатается документ об открытии смены.

  • Внесение денег в кассу для выплаты сдачи: КАССА -> ВНЕСЕНИЕ (по желанию).

Можно произвести процедуру открытия смены комплексно: ДЕЙСТВИЯ -> ОТКРЫТИЕ СМЕНЫ.
В этом случае произойдет комплексный ряд действий: Прием данных с сервера, открытие смены ККМ, запрос на внесение денег в кассу.
Если в результате этой процедуры произошли ошибки программа сообщит об этих ошибках, и предложит произвести действия по отдельности.

Закрытие смены ККМ.
Производится в конце рабочего дня.
Включает в себя процедуры:

  • Сверка итогов с банком (если включен Эквайринг) ЭКВАЙРИНГ -> СВЕРКА ИТОГОВ.

  • Выплата наличности из ККМ: КАССА -> ВЫПЛАТА.

  • Снятие z отчета: КАССА -> Z ОТЧЁТ.

Рекомендуется производить все эти действия в комплексе: ДЕЙСТВИЯ -> ЗАКРЫТИЕ СМЕНЫ.
Если в результате этой процедуры произошли ошибки программа сообщит об этих ошибках, и предложит произвести действия по отдельности.
В конце комплексной процедуры закрытия смены не зависимо от ее результатов - пользователь выходит из программы.

Выбор товара, продажа, печать чека

Что бы произвести продажу нужно сформировать чек.
Чек формируется двумя способами:
Поиск товара через сканер:
Окно ввода кода товара должно быть активным, сканер сканирует продукт и ищет товар с таким ШК в базе, если товар
находится, он заносится в чек, если нет, то выходит окно с предупреждением что товар не найден.
Товар заносится с количеством 1 единица товара, если провести повторно сканером по товару, количество товара увеличится на единицу.
Количество можно скорректировать в ручном режиме, нажать кнопку на товаре в чеке, и ввести необходимое количество.
Внимание Товары могут иметь одинаковые штрихкоды, в таком случае программа предложит ручной выбор из найденных товаров.
Поиск товара в ручном режиме:
Продавец в онкно поиска вводит штрих код товара или его цену.
Удаление выбранного товара производится через окно редактирования количества товара.
Печать чека производится через две кнопки Печать за наличный расчет Печать безналичный расчет.
При печати через безналичный расчет (эквайринг) печатается два чека банка.

Внимание: Поиск по названию отключен, по желанию заказчика.
Внимание: Нулевое количество товара равно как и отрицательное не допускается.
Внимание: Обязательно дождаться конца печати. При зависании, при печати чека, сообщить администратору!!!
Внимание: Отмена оплаты по безналу производится клиентом с пинпада (красная кнопка).
Программа не выполняет эту функцию!!!!

Распоряжения - сообщения

В процессе работы программа производит мониторинг сервера на предмет распоряжений - сообщений для продавца.

Внимание: Распоряжения - сообщения формируются в разрезе продавца.

Распоряжения имеют статус о прочтении, этот статус можно контролировать с сервера.
Если программа (продавец) получает распоряжение - сообщение, на главном меню загорается красная кнопка с лампочкой.
Это символизирует, что в базе программы присутствует не прочитанное (не подтвержденное о прочтении) распоряжение - сообщение.
Продавец ОБЯЗАН в свободное время ознакомиться с распоряжением и зафиксировать это кнопкой о прочтении.
Ознакомление с распоряжением производить через красную кнопку.
Так же можно посмотреть все распоряжение, включая новые, через ДЕЙСТВИЯ -> МОИ СООБЩЕНИЯ.

Внимание: Не откладывайте действия о прочтении, это очень важно в контексте бизнеса.

Заявки

В процессе работы продавца может сложиться ситуация когда нужно принять необходимы меры.
Для этого служит форма заполнения заявки на какую-то работу (замена розетки и тд.).
Для этого продавец должна заполнить форму: ДЕЙСТВИЯ -> СОЗДАТЬ ЗАЯВКУ,
выбрать из выпадающего списка - кому адресована заявка.
Заполнить форму на предмет запрашиваемой работы.
Нажать кнопку:ОТПРАВИТЬ.

Внимание: Не заполненная форма заявки (пустая) отправляться не будет.

Обновление программы

Ничего не вечно, и программы должны обновляться, это связано с устранение ошибок, расширением функционала.
Наша программа в фоновом потоке производит мониторинг сервера на предмет своего обновления, и если
находит, что существует версия старше текущей скачивает ее на компьютер торговой точки и ждет распоряжения пользователя об обновлении.
Существует два варианта показа, что программу можно обновлять.
Вариант АДМИНИСТРАТОР:
На главном меню загорается кнопка ОБНОВЛЕНИЕ.
Горит постоянно не зависимо от режима открытия смены в ККМ.
Администратор, как супер пользователь, сам решает когда обновить программу в разрезе открытой смены ККМ.
Вариант ПРОДАВЕЦ:
Кнопка зажигается только если закрыта смена в ККМ.

Внимание: Обновление программы всегда болезненный процесс.
Что то может пойти не так.
И по этому обновляться рекомендуется когда смена ККМ закрыта и данные о продажах передались на сервер.

Процесс обновления:
Кнопка: МЕНЮ ОБНОВИТЬ, из диалога выбрать обновляемую версию, убедиться, что версия старше текущей (Пример: 0.0.10 старше 0.0.9).
Кнопка диалога: ОБНОВИТЬ, согласится с предупреждением.
В процессе обновления программа сам закроет себя, перекопирует файлы в компьютере и через пару секунд откроется обновленная версия.

Внимание: Обязательно убедиться, что версия обновилась (низ окна программы версия:).
Внимание:Если в результате обновления что то пошло не так ( программа не открылась после обновления).
Звоним АДМИНИСТРАТОРУ.

Откат версии назад.
Может случиться так что версия не подошла по некоторым причинам.
В этом случае можно сделать откат версии на предыдущую.
Это операцию может производить только АДМИНИСТРАТОР.
МЕНЮ АДМИНИСТРАТОР -> ИЗМЕНЕНИЕ ВЕРСИИ ПРОГРАММЫ.

Внимание: Пример : Версия 0.0.2 более ранняя, чем версия 0.0.3.
Администратор должен знать стабильную версию для отката.

Меню продавца: Действия -> Закрытие смены

Производится комплексное закрытие смены

  • Сверка итогов с банком (если включен Эквайринг)

  • Выплат денег из кассы (сумма наличности в ККМ)

  • Печатается Z - отчет

  • Продавец покидает программу

В случае не удачи выводится окно с ошибками и рекомендациями по их устранению

Меню продавца: Действия -> Мои сообщения

Открывается старница сообщений для продавца который их открыл.
Можно посмотреть историю сообщений и их подтвердение о прочитанности.
Не прочитанные сообщения можно прочитать в этом окне а так же подтвердить их прочтение

Меню продавца: Действия -> Обновить данные

Ручное обновление данных, как правило делается комплексно при открытии смены или в ручную в течении тогового дня.

Меню продавца: Действия -> Ошибки

Открывается окно лога ошибок, как правило потребность в этом окне только у администратора при удаленном подключении.

Меню продавца: Действия -> Отправить ошибки на сервер

Отправляется лог ошибок и все настройки программы разработчику.
Предлагается заполнить сопроводительное письмо с уточнением проблемы и причины отправки

Внимание: Отправка происходит в фоновом потоке, рекомендуется 3-4 минуты не закрывать программу после этой процедуры.
